Adding GoDaddy Domain and SSL support to IBM Bluemix

26-03-2016 · 275 words · 2 minute read
Bluemix · Config
Bluemix

Bind the domain on Bluemix

This tutorial assumes that you’ve already purchased a custom domain and know how to navigate on Bluemix console

First declare your custom domain into bluemix: Go through your routes configuration and add a new domain


Standard SSL certificates support only one subdomain + domain: Be careful with wildcard setup

Configure GoDaddy DNS File

Note that the IP points to 75.126.81.68 which is the America’s IBM Bluemix address. If your application is hosted in London you should use: 5.10.124.142

Configure GoDaddy DNS File

Download your certificate files within GoDaddy administration panel

Instal OpenSSL

Now you should install OpenSSL to generate the required certified by running this command:

$ openssl req -new -newkey rsa:2048 -nodes -keyout yourapp.key -out yourapp.csr

$ Country Name (2 letter code) [AU]: BR
$ State or Province Name (full name) [Some-State]: SP
$ Locality Name (eg, city) []: Indaiatuba
$ Organization Name (eg, company) Jungle Badger Inc.
$ Organizational Unit Name (eg, section) []: IT Dept.
$ Common Name (e.g. server FQDN or YOUR name) []: www.yourdomain.com
$ Email Address []: jungle@badger.com

Upload GoDaddy certificates

Inside Bluemix Panel go through the manage routes path and select your previously added custom domain

The first field is reserved for the primary certificate that you’ve downloaded from go Daddy earliear The second field you should insert the openSSL generated certified along with the password that you attached to it. Finally add the intermediate certificate generated by GoDaddy and you should be good to go.

Finishing up

Finish up by uploading the files and saving your changes. Your app shoudl be read to be accessible through your custom domain and SSL enabled
